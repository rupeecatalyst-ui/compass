# Catalyst One — SPR-006D Enterprise Advisory Console (EAC)

**Sprint:** SPR-006D  
**Version:** 13.3.0  
**Status:** Implemented (advisory presentation layer)  
**Date:** July 2026

---

## 1. Enterprise Advisory Console architecture

```
EDE thinks → ERE reasons → DKF supplies knowledge → CHANAKYA communicates → EAC presents & records user response
```

**Never:** Advisory → Execute.

EAC is the central inbox for enterprise recommendations from EDE/ERE (and future Mission Control / Enterprise Analytics). User actions update lifecycle history only.

| Layer | Role |
|-------|------|
| EAC | Review, accept, defer, override, complete, dismiss |
| EDE / ERE | Produce recommendations + reasoning traces |
| CHANAKYA | Executive advisor presentation |
| Dialogue | Lifecycle events |
| ECG / AX | Future templates, priorities, display & lifecycle rules |

Route: `/decisions` (nav: **Advisory Console (EAC)**)

---

## 2. Advisory Lifecycle model

States: `new` → `viewed` → `accepted` | `deferred` | `overridden` | `completed` | `dismissed`

Transitions enforced by ECG-ready `EacLifecycleRules` (scaffold defaults). Every transition writes `EacLifecycleEvent` + enterprise audit.

---

## 3. Override Management model

`EacOverrideRecord`:

- User · Timestamp · Reason · Business Justification · Final Outcome  

Stored in override repository and linked on the advisory card. Appears in Dialogue + audit.

---

## 4. Advisory Card fields

Advisory ID · Opportunity · Customer · Recommendation · Confidence · Advisory Level · Knowledge Packages · Reasoning Trace ID · Generated By · Date & Time · Priority · Status

---

## 5. CHANAKYA presentation

Executive Summary · Business Context · Recommendation · Explanation · Supporting Evidence · Suggested Next Step  

Professional business language only · `neverCommands: true`

---

## 6. ECG / AX integration approach

Adapters resolve (when published on ECG engine `ede`):

- `displayPolicies`  
- `priorityRules`  
- `lifecycleRules`  
- `advisoryTemplates` / `recommendationCategories` (declared on AX contract)

`getEacAdministratorArchitecture()` also declares simulation, template preview, impact analysis, versioning, rollback — **architecture only, no editor UI**.

---

## 7. Dialogue

Lifecycle events: Recommendation Issued / Viewed / Accepted / Deferred / Overridden / Completed / Dismissed.
