## 1. Enterprise Experience Intelligence architecture

```
Recommendation issued (EAC)
   ↓
Experience Record opened (EEI)
   ↓
User Action / Recommendation Outcome
   ↓
Business Outcome (history only)
   ↓
Final Status / Experience Closed
   ↓
Knowledge Feedback reference (admin review later)
```

**Never:** automatic knowledge updates, policy changes, recommendation optimisation, or behavioural scoring.

| Layer | Role |
|-------|------|
| EEI | Capture experience & outcomes |
| EAC | User response to advisories |
| EDE / ERE / DKF | Produce recommendations |
| ECG / AX | Future experience review & knowledge improvement |
| Dialogue | Lifecycle + outcome events |

Route: `/decisions` (nav: **Experience Console (EEI)**)

---

## 2.1 Business Value Traceability (addendum)

Permanent end-to-end linkage on every Experience Record:

```
Recommendation → User Action → Business Outcome → Business Value → Knowledge Package → Reasoning Profile → Decision Trace
```

`EeiBusinessValue` is descriptive only (e.g. faster approval, lower rework, better compliance). **No scores. No analytics.**

`valueTraceability` answers: which recommendation, who acted, what action, what outcome, what value, which knowledge packages, which reasoning profile, which decision trace.

ECG AX also prepares review of high business value / weak knowledge / reasoning profiles needing refinement — architecture only. EEI **never** auto-modifies knowledge, profiles, weighting, workflow, or decision policies.

## 3. Knowledge Feedback architecture

`EeiKnowledgeFeedbackRef` links:

- Knowledge Package(s)  
- Reasoning Profile (when known)  
- Recommendation  
- Outcome  

Administrators may mark `refinementSuggested` — **never auto-published**.

---

## 4. Experience Console

Display order: Recommendation → User Response → Business Outcome → Final Result → Knowledge Package → Reasoning Trace → Experience Record (+ timeline).

Record business outcomes and close experiences from the console.

---

## 5. ECG administrator integration

`getEeiAdministratorArchitecture()` + `resolveEeiReviewProfile()` prepare:

- Experience Review  
- Knowledge Improvement  
- Recommendation Effectiveness  
- Advisory Statistics  
- Simulate / Approve / Publish / Rollback  

**Architecture only.** `autoPublishing: false`.

ECG payload keys: `experienceReview`, `knowledgeImprovement`, `recommendationEffectiveness`, `advisoryStatistics`.

---

## 6. Dialogue

Generated by **Enterprise Experience Intelligence**:

- Recommendation Accepted / Rejected / Overridden / Deferred / Completed  
- Business Outcome Recorded  
- Experience Closed / Experience Recorded  

---

## 7. Recommendations before Mission Control

1. ECG dashboards for accepted/overridden/poor/strong outcome review.  
2. Persist experiences; link Opportunity Workspace business events.  
3. Admin workflow: simulate refinement → approve → publish → rollback.  
4. Mission Control ingest of EEI effectiveness signals (read-only).  
5. Keep ML / auto-learning / auto-policy out of scope.
